# Project: AI Subsidy Application Support System

## 1. プロジェクト概要
助成金（キャリアアップ助成金など）の申請業務を自動化・効率化するWebアプリケーションの開発プロジェクト。
[cite_start]顧客（申請企業）と管理者（TM人事労務コンサルティング）の双方が利用し、書類作成から提出までのプロセスをAIがサポートする [cite: 7, 12, 16]。

## 2. ユーザーロール定義
* **顧客ユーザー（申請企業）**
    * [cite_start]PC操作に不慣れな層も含まれるため、UIは極めてシンプルである必要がある（98%のカバー率を目指す）[cite: 17]。
    * [cite_start]ログイン機能により、自社の申請状況を確認できる [cite: 16]。
* **管理者ユーザー（TM人事労務コンサルティング）**
    * [cite_start]顧客の進捗管理、リマインド設定、ステータス確認を行う [cite: 24]。

## 3. アプリケーション機能要件

### A. 顧客向け機能
1.  **助成金選択とワークフロー表示**
    * [cite_start]ユーザー登録後、申請したい助成金（キャリアアップ助成金、両立支援等助成金など）を選択する [cite: 20]。
    * [cite_start]選択した助成金に応じたタスクリスト（チェックリスト形式）を表示する [cite: 19]。
    * [cite_start]複数の助成金を同時に並行して申請・管理可能とする [cite: 24, 35]。
2.  **書類アップロード・情報入力**
    * [cite_start]企業情報、従業員情報、就業規則などの必要書類をアップロードまたは入力する [cite: 14, 21]。
3.  **AIによる書類チェックと修正提案 (API利用)**
    * [cite_start]**重要:** 高度な推論が必要なため、API（有料）を利用して実装する [cite: 21, 22]。
    * [cite_start]アップロードされた就業規則等が助成金の要件を満たしているか判定する [cite: 21]。
    * [cite_start]不足事項がある場合、AIが「修正案」や「追加すべき条文」を具体的に作成し提示する [cite: 21]。
4.  **申請書類の自動生成**
    * [cite_start]入力された情報とAIの補完に基づき、公式フォーマット（PDF等）に情報を埋め込み、そのまま提出可能な状態で出力する [cite: 14, 22]。

### B. 管理者向け機能
1.  **進捗管理ダッシュボード**
    * [cite_start]顧客ごとの申請ステータスを一覧で確認できる機能 [cite: 24]。
2.  **助成金ルール管理とAI更新**
    * [cite_start]毎年変更される助成金の公募要領やルールを取り込み、AIが差分を抽出してシステム内の判定基準をアップデートする機能 [cite: 15]。
3.  **期限管理と修正**
    * [cite_start]AIが算出した提出期限を管理者が手動で修正・オーバーライドできる機能 [cite: 24]。

### C. 通知・リマインド機能
1.  **自動期限計算**
    * [cite_start]助成金の種類や顧客の状況（例：賃金アップ実施日から6ヶ月経過後など）に基づき、大元のルールソースを参照してAIが自動で提出期限を設定する [cite: 20, 23]。
2.  **リマインド通知**
    * [cite_start]未提出の顧客に対し、期限が近づくと自動でリマインドを行う [cite: 13, 16]。
    * [cite_start]*Future Work:* メールだけでなく、Zoom Phone連携による自動音声架電も検討（技術調査要）[cite: 24]。

## 4. 技術スタック・開発方針
* [cite_start]**プラットフォーム:** Webアプリケーション [cite: 16]。
* [cite_start]**AIモデル利用:** 高度な文書読解・作成を行うため、Gemini API等のLLM API利用を前提とする（従量課金への合意済み）[cite: 21, 22]。
* [cite_start]**開発手法:** 自然言語（日本語）での指示によるノーコード/ローコード開発ツール（Google AI Studio / Claude Code等）を活用し、内製化を目指す [cite: 18, 25]。

## 5. UI/UX デザインガイドライン
* [cite_start]**チェックリスト形式:** 計画届作成・提出フェーズなど、やるべきことを順序立てて表示する [cite: 19]。
* [cite_start]**直感的な操作:** 地方の顧客や高齢の経営者でも迷わず操作できるよう、複雑なメニューを避け、必要な情報を提供すれば入力項目が埋まる設計にする [cite: 16, 17]。
